From the log file, the issue started around the 2nd of September with a java.util.ConcurrentModificationException.   This event continued to repeat until we likely exhausted some resource.   There are several posts about this issue on the internet related to the use of Vector and iterator and the conditions when this exception is raised (listed below).
The version of j2ssh that we are using is 0.2.7 (2003-12-08).  This product is very out of date and there is even a 0.2.9 (2007-09-07) that is available if the j2ssh-maverick-1.5.5 (2016-05-06) could not be used.
The original vendor of j2ssh has a newer version of j2ssh and their next generation version of the API available on the site.  They do not recommend the use of our version of j2ssh anymore:
SSHTOOLS (https://sourceforge.net/projects/sshtools/?source=navbar)
This project now hosts the successor to the original J2SSH API; J2SSH Maverick. This new API was built from the ground up by the original author of J2SSH back in 2003 and has been maintained and available since then exclusively under a commercial license. This API is now available to the community under the LGPL open source license. 
The project continues to host both the original API and legacy applications created around it, however these are now considered deprecated and we do not recommend their use in anyway.
